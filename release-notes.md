## Modelo de Release Notes

### Contexto 
A Release Notes (em tradução livre, "notas de versão") é uma documentação focada em apresentar as principais mudanças, atualizações e novidades de uma nova release, ou seja, de uma nova versão de um produto/serviço.  
Começamos com uma descrição da nova versão. Vale a pena mencionar que existem dois tipos de release, como costumamos falar no mercado:

A major release, que é a release maior e que ocorre quando passamos, por exemplo, da versão 1.0 para a versão 2.0 de um produto;
E a minor release, que é uma release menor, na qual não há uma mudança significativa de uma versão para outra, como da versão 1.0 para 1.1, ou da 1.1 para 1.1.2.

Também apresentamos novas features, caso haja, e as melhorias realizadas. 
Se houve algum ajuste, algum trabalho para facilitar o carregamento do aplicativo ou a usabilidade de alguma maneira, também descrevemos nessa seção.
Além disso, trazemos quais foram os bug fixes, isto é, as correções de bugs e erros que encontramos na aplicação, para que a pessoa usuária saiba que esses erros foram solucionados por nós, enquanto equipe de desenvolvimento.
Existe ainda um campo opcional para falar do que foi descontinuado e o que chamamos de known issues (problemas conhecidos), que são erros que já foram mapeados em versões anteriores do produto, mas que ainda estão em processo de resolução.
